# Security Policy

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| 0.1.x   | :white_check_mark: |

## Reporting a Vulnerability

We take security vulnerabilities seriously. If you discover a security issue, please report it responsibly.

### How to Report

**Please do not open a public issue for security vulnerabilities.**

Instead, report security issues by:

1. **Email**: Send details to the repository maintainers
2. **GitHub Private Vulnerability Reporting**: Use the "Report a vulnerability" feature in the Security tab

### What to Include

- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Suggested fix (if any)
- Any relevant files or screenshots

### Response Timeline

- We will acknowledge receipt of your report within 48 hours
- We aim to provide an initial assessment within 5 business days
- We will keep you updated on our progress throughout the process

### Disclosure Policy

- Please allow us reasonable time to fix the issue before public disclosure
- We will coordinate with you on the public disclosure timeline
- We appreciate responsible disclosure and will credit researchers when possible

## Security Best Practices for Users

- Only extract archives from trusted sources
- Be cautious of archives containing executable files
- Keep Archive Extractor updated to the latest version
